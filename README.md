# Learning Git With Open Data

An ODON course that teaches Git and GitHub to interns and members, built around ODON's
real GitHub organisation and real open data. This repository holds the course design —
the agenda and the six module handouts. It is not where interns do their exercises; that
happens in a separate, private practice repository (see below).

## Who this is for

ODON interns and members with no assumed prior Git experience. The course runs as five
~90-minute sessions, one per week, but can be compressed or stretched to fit a different
format — see [`agenda.md`](agenda.md) for the full rationale, schedule, and facilitator
notes.

## Structure

| Path | Contents |
|---|---|
| [`agenda.md`](agenda.md) | Course overview, module-by-module breakdown, schedule, and practice-repo setup notes — start here if you're facilitating |
| [`handouts/00-setup.md`](handouts/00-setup.md) | Module 0 — Setup & why version control |
| [`handouts/01-git-fundamentals.md`](handouts/01-git-fundamentals.md) | Module 1 — Git fundamentals (working locally) |
| [`handouts/02-branching-merging.md`](handouts/02-branching-merging.md) | Module 2 — Branching & merging |
| [`handouts/03-github-basics.md`](handouts/03-github-basics.md) | Module 3 — GitHub basics: remotes, clone, push & pull |
| [`handouts/04-issues-and-prs.md`](handouts/04-issues-and-prs.md) | Module 4 — Collaboration: Issues, Pull Requests & code review |
| [`handouts/05-open-data.md`](handouts/05-open-data.md) | Module 5 — Working with open data |
| [`data/bike-share-mini.csv`](data/bike-share-mini.csv) | Sample dataset used in Module 5's hands-on exercise |

Each handout is bilingual: English first, German (`Modul N`) below it, kept in the same
file so the two versions can't drift out of sync.

## The practice repository

Hands-on exercises don't happen here — they happen in
**[github.com/odon-at/education-github-playground](https://github.com/odon-at/education-github-playground)**,
a separate, private repo. Ask your mentor to add you as a collaborator before Module 3.
